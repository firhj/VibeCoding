## 二、9周核心知识（深入浅出版）

### Wk01：AI大脑是怎么训练出来的？

**核心比喻：AI就像一个读遍了整个互联网的学生。**

| 阶段 | 做什么 | 比喻 |
|---|---|---|
| **预训练** | 读完数万亿条文本，学会预测下一个词 | 小孩看了几百万本书 |
| **监督微调** | 用优质对话样本教它"好好说话" | 请家教教礼貌和逻辑 |
| **强化学习** | 让它做题，做对了给奖励 | 考试+奖励机制 |
| **提示词工程** | 你教它怎么更好地回答你的问题 | 给它出考题的技巧 |

**关键认知：**
- AI不是"知道答案"，而是"大概率猜对答案"
- AI会"幻觉"——它会一本正经地胡说八道
- "思维链"技术让AI一步一步想问题，大幅提高准确率

---

### Wk02：AI怎么连接外部世界？

**核心比喻：AI本身是个"脑子"，MCP（模型上下文协议）是它的"手和脚"。**

| 概念 | 是什么 | 日常比喻 |
|---|---|---|
| **MCP服务器** | 让AI能调用外部工具的中间层 | 给AI装上USB接口 |
| **工具调用** | AI决定用哪个工具、怎么用 | 指挥官下命令 |
| **RAG** | 让AI查资料再回答，不瞎猜 | 考试允许翻书 |
| **认证（OAuth）** | 安全地授权AI访问你的账号 | 临时门禁卡 |

**关键认知：**
- AI可以连接Jira、GitHub、数据库、网页等真实系统
- MCP统一了AI与工具的沟通方式，不用每个工具写一套代码
- 安全认证让AI只能访问你授权的资源

---

### Wk03：AI IDE（AI编程环境）怎么用？

**核心比喻：Cursor、Devin这类工具就像给程序员配了一个"AI学徒"。**

| 概念 | 解释 | 注意事项 |
|---|---|---|
| **同步代理** | 实时对话，秒级响应（如Copilot） | 适合快速小任务 |
| **异步代理** | 后台运行，几分钟到几小时（如Devin） | 适合复杂大任务 |
| **上下文窗口** | AI能"记住"的文本长度 | 越长越容易"走神" |
| **CLAUDE.md** | 给AI的项目说明书 | 每个项目都要写一份 |

**关键认知：**
- 角色转变：从"写代码的人"变成"管理AI写代码的人"
- 提示词 = 新时代的源代码，应该像代码一样被版本管理
- AI上下文超过10万token后会"分心"，需要人工把关

### Wk04：怎么指挥AI代理工作？

**核心比喻：你从"打工人"变成了"项目经理"，管理多个AI同事。**

| 技巧 | 具体做法 |
|---|---|
| **先想后做** | 先让AI读代码、做计划，确认后再写代码 |
| **测试驱动** | 先写测试，再让AI写代码通过测试 |
| **CLAUDE.md** | 维护好项目规则文件，AI按规则干活 |
| **多代理协作** | 一个AI写代码，另一个AI审查代码 |
| **及时止损** | AI跑偏了就重新开始，别死磕 |

**关键认知：**
- 上下文工程 > 提示词工程
- 好代码是好上下文的副产品
- 回归测试是AI编程的安全网

### Wk05：现代AI工具的七大产品原则

**核心比喻：AI开发工具的进化就像从命令行DOS到图形界面Windows。**

| 主题 | 原则 | 通俗解释 |
|---|---|---|
| 可用性 | 从已知的东西开始 | 用熟悉的界面，不重新学 |
| 可用性 | 人体工学设计 | 5分钟就能上手 |
| 控制力 | 聊天是一等公民 | 用说话代替写代码 |
| 控制力 | 配置灵活 | 新手零配置，老手可微调 |
| 控制力 | 通过MCP集成 | 工具之间能互相对话 |
| 速度 | 快速反馈 | 改了马上能看到效果 |
| 速度 | 完全自主 | AI可以独立干活 |

**关键认知：**
- Vercel V0、Cursor、Warp等工具代表了新的AI原生开发方式
- 代理握方向盘（完全自主）和人在回路（需要审批）是两种模式
- 速度和正确性需要根据场景取舍

### Wk06：AI的安全隐患和上下文腐烂

**核心比喻：AI很聪明但很容易被骗，就像一个记忆力越来越差的天才。**

| 威胁 | 是什么 | 严重程度 |
|---|---|---|
| **提示注入** | 恶意输入骗AI执行危险操作 | 高 |
| **远程代码执行** | 骗AI运行恶意代码 | 极高 |
| **上下文腐烂** | 信息太长AI就记不清、搞混 | 高 |
| **幻觉** | AI自信地给出错误答案 | 中 |

**关键认知：**
- AI代码审查的误报率高达82-86%——每5个"bug"有4个是错的
- 上下文腐烂让AI在长对话中变得不可靠
- 代码执行必须在沙箱中运行，不能直接信任AI的输出

---

### Wk07：AI代码审查的黄金法则

**核心比喻：AI是完美的"自动质检员"，但它不懂业务逻辑。**

| 审查领域 | AI擅长吗 | 人类擅长吗 |
|---|---|---|
| 简单bug | 擅长 | 一般 |
| 性能问题 | 擅长 | 一般 |
| 安全漏洞 | 擅长 | 一般 |
| 业务逻辑 | 不擅长 | 擅长 |
| 部落知识 | 不擅长 | 擅长 |
| 架构设计 | 不擅长 | 擅长 |

**关键认知：**
- 代码审查的价值层次：心智对齐 > 方案验证 > 设计讨论 > 找bug > 代码风格
- AI调优后在"促使必要代码变更"上已经和人类一样有效
- 调少规则反而更好——减少17条低价值规则，接受率从54%提升到80%+

### Wk08：AI自动化构建应用

**核心比喻：以前需要一个团队花几个月写的代码，现在跟AI聊天几分钟就能生成。**

| 旧方式 | 新方式 |
|---|---|
| 手写LAMP/MERN全栈 | 提示AI生成React/Next.js组件 |
| 手动配置数据库、API、部署 | AI一键处理配置 |
| 几个月出MVP | 几天甚至几小时出Demo |
| 专业团队 | 一个人 + AI |

**AI生成可靠代码的秘密：**
- 流操作：在代码到达用户前自动修复错误
- 自动修复器：小型AI检测bug，大型AI修复bug
- 专用模型比通用模型好15%（V0 1.5: 93.87% vs Claude 4: 78%）

**关键认知：**
- 开发速度被加速后，瓶颈从"做出来"变成了"想清楚"
- 好的想法和清晰的规格说明比写代码更重要
- 价值从执行转移到了验证和策略

---

### Wk09：AI代理部署后怎么运维？

**核心比喻：以前运维像消防员救火，现在AI运维像智能预警系统。**

| 阶段 | 运维方式 | 特点 |
|---|---|---|
| 传统 | 系统管理员手工操作 | 贵、慢、容易出错 |
| SRE时代 | 工程师用代码解决运维问题 | 自动化、有错误预算 |
| AI原生 | AI代理自主诊断和修复 | 秒级响应、持续学习 |

**关键认知：**
- "希望不是一种策略"——大型系统必须有自动化监控
- SRE的核心思想：运维工作不超过50%，另50%做持久性工程
- AI原生运维不是"AI辅助"，而是AI独立接管诊断和修复
- 最大的挑战：给AI代理这么大的控制权，安全怎么办？

## 三、初学者最该记住的5个概念

1. **提示词工程**：学会和AI说话，就是学会编程
2. **上下文工程**：AI的"工作记忆"有限，你需要帮它整理好信息
3. **人在回路中**：AI做执行，人做决策和判断
4. **测试驱动**：永远用测试来验证AI写的代码对不对
5. **安全第一**：AI会犯错、会被骗，不能盲目信任它的输出

## 四、CS146S揭示的核心趋势

1. **角色转变**：程序员从"编码者"变成"代理管理者"
2. **价值转移**：从"写代码的速度"变成"想清楚要写什么"
3. **门槛降低**：不会代码也能用AI做出应用
4. **安全挑战**：AI能力越强，安全风险越大
5. **持续学习**：AI工具在快速进化，半年后今天的最佳实践可能就过时了
